## 1. 系统概述

### 1.1 系统简介

IOT 管理系统是一个功能完善的物联网设备管理平台，采用微服务架构设计，支持多种定位技术和实时监控功能。

### 1.2 主要功能

| 功能模块 | 说明 |
|---------|------|
| **地图与定位** | 支持高德地图（RTK）、自定义地图（UWB）等多种地图类型 |
| **设备管理** | 设备标记的增删改查，支持类型和标签分类 |
| **基站管理** | UWB 基站配置与管理 |
| **电子围栏** | 多边形围栏设置，进出警报 |
| **距离监控** | 设备间安全距离监控，超限警报 |
| **用户管理** | 多用户权限控制（普通用户、管理员、超级管理员） |

### 1.3 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                        客户端层                              │
│              Web 前端 (Vue 3 + TypeScript)                   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       API 网关 (端口 8000)                    │
│                    统一入口，JWT 认证，路由转发                 │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│  用户服务 8001 │    │  地图服务 8002 │    │ MQTT监控 8003 │
│  (用户认证)    │    │  (地图/围栏)   │    │  (消息处理)    │
└───────────────┘    └───────────────┘    └───────────────┘
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              ▼
              ┌───────────────────────────────┐
              │       PostgreSQL + PostGIS    │
              │           MongoDB              │
              │        MQTT Broker             │
              └───────────────────────────────┘
```

### 1.4 服务端口

| 服务 | 端口 | 说明 |
|-----|------|------|
| 前端界面 | 80, 3000 | Web 访问入口 |
| API 网关 | 8000 | 统一 API 入口 |
| 用户服务 | 8001 | 用户认证管理 |
| 地图服务 | 8002 | 地图、围栏管理 |
| MQTT 监控 | 8003 | 消息订阅处理 |
| 标记服务 | 8004 | 设备标记管理 |
| PostgreSQL | 5432 | 主数据库 |
| MongoDB | 27017 | 位置历史存储 |
| MQTT Broker | 1883/8083 | 消息代理 |

## 2. 快速开始

### 2.1 环境要求

- 现代浏览器（Chrome、Firefox、Edge、Safari）
- Docker 和 Docker Compose（用于部署）
- MQTT 客户端工具（可选，用于设备模拟）

### 2.2 访问系统

1. 启动所有服务：
```bash
docker-compose up -d
```

2. 打开浏览器访问：`http://localhost`

3. 进入登录页面

### 2.3 默认账户

| 角色 | 用户名 | 密码 | 权限 |
|-----|--------|------|------|
| 管理员 | admin | admin | 全部功能 |

### 2.4 首次登录

1. 输入用户名和密码
2. 点击登录按钮
3. 登录成功后自动跳转到首页

## 5. 地图与定位

### 5.1 RTK 实时地图

RTK（实时动态定位）是一种高精度 GPS 定位技术，适用于室外环境。

**功能说明：**
- 在高德地图上显示设备实时位置
- 支持多设备同时显示
- 显示设备最新坐标信息
- 实时更新设备位置

**操作步骤：**
1. 点击顶部菜单 **地图 → 实时 RTK 地图**
2. 地图加载后显示当前区域
3. 在线设备以标记形式显示在地图上
4. 点击设备标记可查看详细信息

### 5.2 UWB 实时地图

UWB（超宽带）是一种室内定位技术，适用于室内环境。

**功能说明：**
- 在自定义地图上显示设备位置
- 支持上传平面图作为底图
- 坐标系精确配置
- 室内定位精度高

**操作步骤：**
1. 点击顶部菜单 **地图 → 实时 UWB 地图**
2. 系统加载已配置的自定义地图
3. 设备以坐标点形式显示
4. 支持轨迹绘制功能

### 5.3 地图总览

综合展示页面，可同时查看 RTK 和 UWB 设备的实时位置。

**访问路径：** 地图 → 地图总览

### 5.4 历史轨迹

查看设备历史移动轨迹。

**操作步骤：**
1. 进入 RTK 地图或 UWB 地图
2. 选择要查看的设备
3. 系统显示该设备的移动轨迹
4. 支持时间范围筛选

## 6. 基站管理

基站是 UWB 定位系统的参考点设备，用于确定移动设备的精确位置。

### 6.1 查看基站列表

**访问路径：** 地图 → UWB 基站管理

**显示信息：**
- 基站 ID
- 基站名称
- 基站类型
- 坐标位置（X, Y）
- 状态

### 6.2 添加基站

1. 进入基站管理页面
2. 点击"添加基站"按钮
3. 填写基站信息：
   - 基站名称
   - 基站类型
   - X 坐标
   - Y 坐标
4. 点击"保存"

### 6.3 编辑基站

1. 在基站列表中找到要编辑的基站
2. 点击"编辑"按钮
3. 修改基站信息
4. 点击"保存"

### 6.4 删除基站

1. 在基站列表中找到要删除的基站
2. 点击"删除"按钮
3. 确认删除操作

### 6.5 基站布局建议

- 基站应均匀分布在定位区域内
- 建议在区域角落和边缘布置
- 避免基站之间距离过近或过远
- 室外 RTK 定位不需要基站

## 8. 距离设置

距离设置用于配置设备间的安全距离，当设备间距离小于安全距离时触发警报。

### 8.1 设备对距离配置

为两个特定设备设置安全距离。

**访问路径：** 电子标记 → 距离设置

**操作步骤：**
1. 选择第一个设备
2. 选择第二个设备
3. 输入安全距离值（单位：米）
4. 点击"保存"

### 8.2 笛卡尔积配置

为同一类型的设备批量设置安全距离。

**操作步骤：**
1. 选择设备类型
2. 输入默认安全距离
3. 系统自动为该类型下所有设备对设置距离

### 8.3 批量组合配置

手动选择多个设备，批量设置它们之间的安全距离。

**操作步骤：**
1. 选择要配置的设备列表
2. 输入安全距离值
3. 确认配置

### 8.4 距离配置优先级

1. **设备对距离**：针对特定两个设备的自定义距离（最高优先级）
2. **类型默认距离**：同一类型设备的默认安全距离
3. **跳过检查**：设置为 -1 表示不检查该设备对

## 9. 电子围栏

电子围栏是在地图上划定的虚拟安全区域，当设备进入或离开围栏时触发警报。

### 9.1 围栏类型

| 类型 | 说明 |
|-----|------|
| 室内围栏 | 室内定位系统使用的围栏 |
| 室外围栏 | 室外 RTK 定位系统使用的围栏 |

### 9.2 围栏列表

**访问路径：** 地图 → 地图设置 → 电子围栏

**显示信息：**
- 围栏 ID
- 围栏名称
- 围栏类型
- 描述
- 关联设备
- 操作

### 9.3 创建围栏

1. 进入围栏管理页面
2. 点击"创建围栏"按钮
3. 填写围栏信息：

| 字段 | 说明 |
|-----|------|
| 围栏名称 | 围栏标识名称 |
| 围栏类型 | 室内或室外 |
| 坐标点 | 围栏边界的顶点坐标 |
| 描述 | 围栏说明 |

4. 在地图上绘制围栏边界
5. 点击"保存"

### 9.4 绘制围栏

1. 选择地图上的点作为围栏顶点
2. 依次点击形成围栏边界
3. 点击完成绘制
4. 系统自动计算围栏面积

### 9.5 编辑围栏

1. 在围栏列表中找到要编辑的围栏
2. 点击"编辑"按钮
3. 修改围栏信息或边界
4. 点击"保存"

### 9.6 删除围栏

1. 在围栏列表中找到要删除的围栏
2. 点击"删除"按钮
3. 确认删除操作

### 9.7 围栏警报规则

- **进入警报**：设备进入围栏区域时触发
- **离开警报**：设备离开围栏区域时触发
- **持续警报**：设备在围栏内持续触发（可配置）

## 11. 警报系统

### 11.1 警报类型

| 类型 | 触发条件 |
|-----|---------|
| 距离警报 | 两个设备间距离小于安全距离 |
| 围栏进入警报 | 设备进入电子围栏区域 |
| 围栏离开警报 | 设备离开电子围栏区域 |

### 11.2 距离警报

当两个设备的距离小于设定的安全距离时触发。

**触发流程：**
```
设备位置更新 → 计算设备间距离 → 判断是否小于安全距离 → 触发警报
```

### 11.3 围栏警报

当设备进入或离开围栏区域时触发。

**触发流程：**
```
设备位置更新 → 检查围栏状态 → 判断进入/离开 → 触发警报
```

### 11.4 警报限流

系统内置警报限流机制，防止警报风暴：
- 每个设备最多每秒 2 次警报
- 相同状态的警报会被合并

### 11.5 查看警报

**访问路径：** 导航菜单中的警报入口

**显示信息：**
- 警报时间
- 警报类型
- 涉及设备
- 警报详情

## 13. 故障排查

### 13.1 常见问题

#### 1. 登录失败
**可能原因：**
- 用户名或密码错误
- 账户被禁用

**解决方案：**
- 确认用户名密码正确
- 联系管理员检查账户状态

#### 2. 地图不显示
**可能原因：**
- 高德地图 Key 未配置
- 网络连接问题

**解决方案：**
- 检查 .env 文件中的 `VITE_AMAP_KEY` 配置
- 确认网络连接正常

#### 3. 设备位置不更新
**可能原因：**
- 设备离线
- MQTT 连接失败

**解决方案：**
- 检查设备状态
- 重启 MQTT 服务

#### 4. 警报不触发
**可能原因：**
- 距离/围栏配置错误
- Warning 服务未启动

**解决方案：**
- 检查距离设置和围栏配置
- 检查 Warning 服务日志

#### 5. 无法访问系统
**可能原因：**
- Docker 服务未启动
- 端口被占用

**解决方案：**
- 执行 `docker-compose up -d`
- 检查端口占用：`netstat -ano | findstr :80`

### 13.2 日志排查

查看各服务日志定位问题：
```bash
# 查看 API 网关日志
docker-compose logs api-gateway

# 查看用户服务日志
docker-compose logs user-service

# 查看地图服务日志
docker-compose logs map-service

# 查看 MQTT 服务日志
docker-compose logs mosquitto
```

### 13.3 网络检查

```bash
# 检查端口占用
netstat -ano | findstr :8000

# 检查 Docker 容器网络
```

## 14. API 参考

### 14.1 API 概述

系统提供 RESTful API 接口，通过 API 网关统一访问。

**基础 URL：** `http://localhost:8000`

### 14.2 认证说明

API 使用 JWT Token 进行认证。

**请求头：**
```
Authorization: Bearer <token>
```

**登录获取 Token：**
```bash
POST /api/v1/users/token
Content-Type: application/json

{
  "username": "admin",
  "password": "admin"
}
```

**响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_at": "2024-01-01T00:00:00Z"
  }
}
```

### 14.3 统一响应格式

**成功响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

**错误响应：**
```json
{
  "code": 400,
  "message": "错误信息",
  "details": "详细错误说明"
}
```
