## 1. 系统概述

### 1.1 管理系统简介

管理系统是 IOT 管理系统的配置管理端，面向负责系统配置、设备管理、围栏设置的运维人员和管理员。

### 1.2 主要功能

| 功能 | 说明 |
|-----|------|
| **用户管理** | 用户登录、密码修改、权限管理 |
| **设备管理** | 设备的增删改查，类型和标签管理 |
| **基站管理** | UWB 基站配置与管理 |
| **地图设置** | 自定义地图上传、坐标系配置 |
| **电子围栏** | 围栏创建、编辑、删除，警报规则配置 |
| **距离设置** | 设备间安全距离配置 |
| **运维管理** | 服务部署、数据备份、日志查看 |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      管理系统客户端                          │
│                   (Vue 3 + TypeScript)                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      API 网关 (端口 8000)                    │
│                 统一入口，JWT 认证，路由转发                   │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│  用户服务 8001 │    │  地图服务 8002 │    │ MQTT监控 8003 │
│  (用户认证)    │    │  (地图/围栏)   │    │  (消息处理)    │
└───────────────┘    └───────────────┘    └───────────────┘
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              ▼
              ┌───────────────────────────────┐
              │       PostgreSQL + PostGIS    │
              │           MongoDB              │
              │        MQTT Broker             │
              └───────────────────────────────┘
```

### 1.4 服务端口

| 服务 | 端口 | 说明 |
|-----|------|------|
| 前端界面 | 80, 3000 | Web 访问入口 |
| API 网关 | 8000 | 统一 API 入口 |
| 用户服务 | 8001 | 用户认证管理 |
| 地图服务 | 8002 | 地图、围栏管理 |
| MQTT 监控 | 8003 | 消息订阅处理 |
| 标记服务 | 8004 | 设备标记管理 |
| PostgreSQL | 5432 | 主数据库 |
| MongoDB | 27017 | 位置历史存储 |
| MQTT Broker | 1883/8083 | 消息代理 |

### 5.5 坐标系说明

UWB 定位使用平面直角坐标系：

```
Y
↑
│    ┌─────────┐
│    │  地图   │
│    │        │
│    └─────────┘
│         → X
原点 (0, 0)
```

- X 坐标：从左到右增加
- Y 坐标：从下到上增加
- 单位：与实际距离单位一致（如厘米）

## 7. 距离设置

距离设置用于配置设备间的安全距离，当设备间距离小于安全距离时触发警报。

### 7.1 距离设置页面

**访问路径：** 电子标记 → 距离设置

### 7.2 设备对距离配置

为两个特定设备设置安全距离。

**操作步骤：**
1. 选择第一个设备
2. 选择第二个设备
3. 输入安全距离值（单位：米）
4. 点击"保存"

**页面截图：**
```markdown
![距离设置页面](./screenshots/admin_distance_settings.png)
```
*图 7-1：距离设置页面*

### 7.3 笛卡尔积配置

为同一类型的设备批量设置安全距离。

**操作步骤：**
1. 选择设备类型
2. 输入默认安全距离
3. 系统自动为该类型下所有设备对设置距离

### 7.4 批量组合配置

手动选择多个设备，批量设置它们之间的安全距离。

**操作步骤：**
1. 选择要配置的设备列表
2. 输入安全距离值
3. 确认配置

### 7.5 距离配置优先级

| 优先级 | 类型 | 说明 |
|-------|------|------|
| 1 | 设备对距离 | 针对特定两个设备的自定义距离 |
| 2 | 类型默认距离 | 同一类型设备的默认安全距离 |
| -1 | 跳过检查 | 设置为 -1 表示不检查该设备对 |

## 8. 运维指南

### 8.1 Docker 部署

#### 启动所有服务
```bash
docker-compose up -d
```

#### 查看服务状态
```bash
docker-compose ps
```

#### 查看服务日志
```bash
docker-compose logs -f [服务名]
```

#### 停止所有服务
```bash
docker-compose down
```

#### 重启服务
```bash
docker-compose restart [服务名]
```

### 8.2 服务管理

| 操作 | 命令 |
|-----|------|
| 启动 | `docker-compose up -d` |
| 停止 | `docker-compose down` |
| 重启 | `docker-compose restart` |
| 查看日志 | `docker-compose logs -f` |
| 进入容器 | `docker exec -it [容器名] /bin/bash` |

### 8.3 数据备份

#### PostgreSQL 备份
```bash
docker exec [容器名] pg_dump -U postgres iot_manager_db > backup.sql
```

#### PostgreSQL 恢复
```bash
docker exec -i [容器名] psql -U postgres iot_manager_db < backup.sql
```

#### MongoDB 备份
```bash
docker exec [容器名] mongodump --username admin --password admin --out /backup
```

#### MongoDB 恢复
```bash
docker exec [容器名] mongorestore --username admin --password admin /backup
```

### 8.4 日志管理

```bash
# 查看所有服务日志
docker-compose logs

# 查看特定服务日志
docker-compose logs -f api-gateway

# 查看最近 100 行日志
docker-compose logs --tail=100
```

## 9. 故障排查

### 9.1 常见问题

#### 1. 登录失败
**可能原因：**
- 用户名或密码错误
- 账户被禁用

**解决方案：**
- 确认用户名密码正确
- 联系管理员检查账户状态

#### 2. 地图不显示
**可能原因：**
- 高德地图 Key 未配置
- 网络连接问题
- 地图未正确上传

**解决方案：**
- 检查 .env 文件中的 `VITE_AMAP_KEY` 配置
- 确认网络连接正常
- 检查地图配置是否正确

#### 3. 设备位置不更新
**可能原因：**
- 设备离线
- MQTT 连接失败
- 设备未配置 MQTT 持久化

**解决方案：**
- 检查设备状态
- 重启 MQTT 服务
- 确认设备配置正确

#### 4. 警报不触发
**可能原因：**
- 距离/围栏配置错误
- Warning 服务未启动
- 设备未在监控范围内

**解决方案：**
- 检查距离设置和围栏配置
- 检查 Warning 服务日志
- 确认设备位置数据正常

#### 5. 无法访问系统
**可能原因：**
- Docker 服务未启动
- 端口被占用

**解决方案：**
- 执行 `docker-compose up -d`
- 检查端口占用：`netstat -ano | findstr :80`

### 9.2 日志排查

查看各服务日志定位问题：

```bash
# 查看 API 网关日志
docker-compose logs api-gateway

# 查看用户服务日志

# 查看地图服务日志
docker-compose logs map-service

# 查看 MQTT 服务日志

```

### 9.3 网络检查

```bash

docker network ls

# 检查容器网络连接

```

## 10. API 参考

### 10.1 API 概述

系统提供 RESTful API 接口，通过 API 网关统一访问。

**基础 URL：** `http://localhost:8000`

### 10.2 认证说明

API 使用 JWT Token 进行认证。

**请求头：**
```
Authorization: Bearer <token>
```

**登录获取 Token：**
```bash
POST /api/v1/users/token
Content-Type: application/json

{
  "username": "admin",
  "password": "admin"
}
```

**响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_at": "2024-01-01T00:00:00Z"
  }
}
```

### 10.3 统一响应格式

**成功响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

**错误响应：**
```json
{
  "code": 400,
  "message": "错误信息",
```
