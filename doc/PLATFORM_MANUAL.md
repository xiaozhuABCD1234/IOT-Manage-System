## 1. 系统概述

### 1.1 展示平台简介

展示平台是 IOT 管理系统的监控查看端，面向需要实时监控设备状态、查看位置信息的一线操作人员。

### 1.2 主要功能

| 功能 | 说明 |
|-----|------|
| **实时地图** | 在地图上查看设备当前位置，支持 RTK（室外）和 UWB（室内）两种定位模式 |
| **地图总览** | 综合展示所有设备的实时位置 |
| **历史轨迹** | 查看设备历史移动轨迹 |
| **设备状态** | 查看设备在线/离线状态 |
| **警报查看** | 查看设备触发的警报信息 |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      展示平台客户端                          │
│                   (Vue 3 + TypeScript)                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      API 网关 (端口 8000)                    │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│  用户服务 8001 │    │  地图服务 8002 │    │ MQTT监控 8003 │
└───────────────┘    └───────────────┘    └───────────────┘
                              │
                              ▼
              ┌───────────────────────────────┐
              │       PostgreSQL + PostGIS    │
              │           MongoDB              │
              │        MQTT Broker             │
              └───────────────────────────────┘
```

## 4. 实时地图

实时地图用于在地图上查看设备的当前位置。

### 4.1 RTK 实时地图

RTK（实时动态定位）是一种高精度 GPS 定位技术，适用于室外环境。

**功能说明：**
- 在高德地图上显示设备实时位置
- 支持多设备同时显示
- 显示设备最新坐标（经纬度）
- 实时更新设备位置

**访问路径：** 地图 → 实时 RTK 地图

**操作说明：**
1. 进入 RTK 实时地图页面
2. 地图加载后显示当前区域
3. 在线设备以标记形式显示在地图上
4. 标记颜色可能表示不同状态：
   - 绿色：设备在线，正常
   - 红色：设备触发警报
5. 点击设备标记可查看详细信息：
   - 设备名称
   - 设备类型
   - 当前坐标
   - 最后更新时间

**页面截图：**
```markdown
![RTK实时地图](./screenshots/platform_rtk_map.png)
```
*图 4-1：RTK实时地图页面*

### 4.2 UWB 实时地图

UWB（超宽带）是一种室内定位技术，适用于室内环境。

**功能说明：**
- 在自定义地图（平面图）上显示设备位置
- 室内定位精度高
- 支持多设备同时显示
- 实时更新设备位置

**访问路径：** 地图 → 实时 UWB 地图

**操作说明：**
1. 进入 UWB 实时地图页面
2. 系统加载已配置的自定义地图（平面图）
3. 设备以坐标点形式显示在地图上
4. 基站位置也会显示在地图上
5. 点击设备标记可查看详细信息

**页面截图：**
```markdown
![UWB实时地图](./screenshots/platform_uwb_map.png)
```
*图 4-2：UWB实时地图页面*

**访问路径：** 地图 → 地图总览

- 同时显示室内外所有设备

## 5. 历史轨迹

查看设备历史移动轨迹，用于回放设备运动路径。

### 5.1 功能说明

- 显示设备历史移动路径
- 支持时间范围筛选
- 支持多设备轨迹对比
- 轨迹数据存储在 MongoDB 中

### 5.2 查看轨迹

**访问路径：** 地图 → 实时 RTK 地图 / 实时 UWB 地图

**操作步骤：**
1. 进入地图页面
2. 在设备列表或设备选择器中选择要查看的设备
3. 选择时间范围
4. 系统显示该设备的移动轨迹
5. 轨迹以线段或点序列形式呈现

### 5.3 轨迹显示说明

| 元素 | 说明 |
|-----|------|
| 轨迹线 | 连接设备历史位置的线段 |
| 轨迹点 | 设备经过的位置标记 |
| 方向箭头 | 表示移动方向（可选） |
| 时间标签 | 显示各点的时间信息 |

## 6. 设备状态监控

设备状态监控页面用于查看所有设备的在线/离线状态。

### 6.1 标记状态

**访问路径：** 电子标记 → 标记状态

**显示信息：**

| 字段 | 说明 |
|-----|------|
| 设备名称 | 设备显示名称 |
| 设备类型 | 设备所属分类 |
| 在线状态 | 在线/离线 |
| 最后在线时间 | 最近一次收到数据的时间 |
| 最后位置 | 设备最后已知位置 |

**页面截图：**
```markdown
![标记状态页面](./screenshots/platform_mark_status.png)
```
*图 6-1：设备标记状态页面*

### 6.2 状态说明

| 状态 | 说明 | 标识 |
|-----|------|-----|
| 在线 | 设备当前正在发送数据 | 绿色 |
| 离线 | 设备超过一定时间未发送数据 | 灰色/红色 |

### 6.3 筛选功能

支持按以下条件筛选设备：

- **设备类型**：按设备分类筛选
- **设备标签**：按标签筛选
- **在线状态**：只看在线/离线设备

### 6.4 状态刷新

- 页面自动定时刷新设备状态
- 也支持手动刷新
- 实时数据通过 MQTT 推送更新

## 7. 警报查看

警报页面用于查看设备触发的各类警报信息。

### 7.1 警报类型

| 类型 | 触发条件 | 说明 |
|-----|---------|------|
| 距离警报 | 两个设备间距离小于安全距离 | 设备距离过近时触发 |
| 围栏进入警报 | 设备进入电子围栏区域 | 不应进入的区域 |
| 围栏离开警报 | 设备离开电子围栏区域 | 离开安全区域 |

### 7.2 警报显示

**访问路径：** 导航菜单中的警报入口

**显示信息：**

| 字段 | 说明 |
|-----|------|
| 警报时间 | 警报触发的时间 |
| 警报类型 | 距离/围栏进入/围栏离开 |
| 涉及设备 | 触发警报的设备 |
| 警报详情 | 具体信息，如距离值、围栏名称等 |
| 处理状态 | 未处理/已处理 |

**页面截图：**
```markdown
![警报页面](./screenshots/platform_warning.png)
```
*图 7-1：警报查看页面*

### 7.3 警报颜色标识

| 状态 | 颜色 | 说明 |
|-----|------|------|
| 距离警报 | 橙色/黄色 | 设备距离过近 |
| 围栏进入警报 | 红色 | 设备进入禁区 |
| 围栏离开警报 | 蓝色 | 设备离开安全区 |

### 7.4 警报处理

- 查看警报详情
- 标记警报为已处理
- 持续未处理的警报可能持续显示

### 7.5 警报说明

系统内置警报限流机制，防止警报风暴：
- 每个设备最多每秒 2 次警报
- 相同状态的警报会被合并
- 避免大量重复警报干扰
